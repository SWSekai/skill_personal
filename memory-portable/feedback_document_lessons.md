當遇到實戰中發現的注意事項、踩坑經驗、或細微修正時，應主動評估是否值得寫入專案指南或手冊文件中。

**Why:** 這類經驗容易遺忘，但對未來部署同類專案非常有價值。寫入文件後可透過 /pack 帶到新專案，避免重複踩坑。

**How to apply:**
- 每次修正 CI/CD、部署、配置等問題後，評估該經驗是否具通用性
- 如果是，主動提議寫入 `.local/docs/` 下的對應指南文件（如 k8s-deployment-guide、cicd-troubleshooting 等）
- 文件格式：問題描述 → 原因 → 解決方式，方便日後查閱
- 確保這些文件在 /pack 時會被收集到 ai-context 中
