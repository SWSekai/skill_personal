## Markdown Interactive Collaboration Protocol

### Usage

```
/md-collab <topic> [output_path]
```

- `<topic>`：決策主題（必填），例如「API 權限配置」「功能開關」「遷移計畫」
- `[output_path]`：輸出檔案路徑（選填），預設為專案工作目錄下的適當位置

---

### Step 1: 分析主題與蒐集資料

根據 `<topic>` 決定需要哪些資訊：

1. **掃描相關程式碼**：找出與主題相關的現有實作、設定檔、API 端點
2. **識別決策項目**：列出所有需要使用者決定的項目
3. **整理現狀**：記錄每個項目的目前狀態作為參考欄位

---

### Step 2: 生成互動式 Markdown 文件

在 `output_path` 建立結構化的 Markdown 檔案，遵循以下格式規範：

#### 2-1. 檔案結構

```markdown
# {主題名稱}

{使用說明：告知使用者如何填寫，勾選規則}

---

## 一、{分類名稱}

| # | 項目 | 現狀 | [ ] 選項A | [ ] 選項B | [ ] 選項C |
|---|------|:----:|:---------:|:---------:|:---------:|
| 1 | 項目名稱 | 目前值 | [x] | [ ] | [ ] |
| 2 | 項目名稱 | 目前值 | [ ] | [x] | [ ] |

---

## 二、{分類名稱}
...
```

#### 2-2. 格式規範

- **核取方塊**：使用 `[ ]`（未勾選）和 `[x]`（已勾選）
- **預填建議值**：根據分析結果預先勾選建議選項，降低使用者負擔
- **現狀欄位**：顯示目前實際狀態，方便對比
- **編號連續**：所有項目全域編號，方便口頭討論時引用（如「第 17 項改成 B」）
- **分類分組**：相關項目歸入同一區塊，區塊間以 `---` 分隔
- **表頭說明**：每個選項欄位需有清楚的語義（不要只用 A/B/C）

#### 2-3. 支援的互動模式

根據主題性質選擇最適合的模式：

**模式 A — 單選矩陣**（如權限等級配置）
```markdown
| # | 項目 | [ ] 低 | [ ] 中 | [ ] 高 |
```
每列僅勾選一個選項，表示選定的等級/類別。

**模式 B — 多選清單**（如功能開關、遷移項目）
```markdown
| # | 項目 | 說明 | [ ] 啟用 |
```
可勾選多個項目，表示啟用/選取。

**模式 C — 自由填寫**（如參數配置、名稱對照）
```markdown
| # | 項目 | 現狀 | 新值（請填寫） |
```
使用者直接在表格中填入文字值。

**模式 D — 混合模式**
上述模式可混合使用，不同區塊採用不同模式。

---

### Step 3: 引導使用者編輯

產生檔案後，向使用者說明：

1. 檔案位置與開啟方式
2. 填寫規則（例：「僅勾選最低允許等級」「可多選」）
3. 預填值的含義（「已根據現狀預填建議，可直接修改」）
4. 完成後告知 Claude 繼續

**輸出範例**：
```
已建立互動式配置表：`<path>`

請開啟檔案，依照表頭說明勾選各項目的設定值。
預填值為根據現狀的建議配置，可自行調整。
完成後回覆「OK」或「完成」，我會讀取並實作。
```

---

### Step 4: 讀取使用者決策

使用者回覆完成後：

1. **讀取檔案**：重新讀取 Markdown 檔案
2. **解析勾選結果**：提取每個項目的選定值
3. **差異比對**：列出與現狀不同的變更項目
4. **生成變更摘要**：以表格形式呈現「項目 / 現狀 → 新值」

**摘要範例**：
```markdown
## 變更摘要

| # | 項目 | 現狀 | → 新值 |
|---|------|------|--------|
| 3 | 標註圖片選單 | user | → view |
| 17 | 上傳資料集 | 無認證 | → user |

共 2 項變更，44 項維持不變。
```

---

### Step 5: 確認與實作

1. **向使用者展示變更摘要**，確認無誤
2. **逐項實作變更**：根據決策修改對應的程式碼/設定
3. **交叉驗證**：實作完成後回頭比對原始決策表，確保無遺漏

---

### 設計原則

1. **降低認知負擔**：預填合理建議值，使用者只需調整差異
2. **全域編號**：跨區塊連續編號，方便口頭快速引用
3. **現狀可見**：每項都附帶目前狀態，使用者不需另外查找
4. **結構化輸出**：表格格式讓決策一目了然，也方便後續程式化解析
5. **可追溯**：檔案保留作為決策紀錄

Arguments: $ARGUMENTS (topic and optional output path)
